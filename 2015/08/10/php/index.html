然后我大概发现是rewrite模块的问题。php中好像这个模块不开.htaccess这个文件是无法生效的。然后我就在找怎么样才能把rewrite模块打开。在/etc/apache2/里面找了好久都没有找到http.d文件，我猜想我是用apt-get install xxx安装的，所以和编译出来的文件夹目录不一样 可以用tree命令查看哦。

后来晚上请教豪哥，豪哥也半天没搞出来。跟我下午时查找bug时的情形差不多。我们对比了可用的配置bbs和我们要配置的reg文件。又发现bbs目录下同样也有.htaccess文件。但是bbs中我们在index.php中写入的phpinfo();同样没有rewrite模块。但是确实是开启了的。可用！通过查看a2ensite和对sites-available sites-enabled 的link连接判断无误。

于是我们把目光转向.htaccess这个文件。虽然它是个没有后缀的文件 但是通过file -l命令还是获取到了.htaccess文件的字符串编码。也就是格式问题。我所采用的是utf8 BOM格式，它会在文件前面自动加上一些东西，这样就会造成.htaccess的失效。采用utf8 无BOM格式的就不会有问题。这真是黑。
